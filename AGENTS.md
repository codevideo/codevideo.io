# AGENTS.md

This repo is expected to have SEO + agent-readiness primitives (or be moving toward them) including metadata, JSON-LD, OG images, robots/sitemap and `llms.txt`

When making changes related to SEO, crawling, indexing, social previews, or AI/agent access, apply these rules.

**Architecture note**: This is a **Gatsby** project (not Next.js). SEO metadata is handled via `react-helmet` in `src/components/SEO.tsx`, not via Next.js `metadata` exports.

## Canonical + URLs

- Keep the canonical origin in one place: `gatsby-config.ts` → `siteMetadata.siteUrl` (`https://codevideo.io`).
- If the domain changes, update all of:
  - `siteMetadata.siteUrl` in `gatsby-config.ts`
  - the `siteUrl` constant used in `src/components/SEO.tsx` for canonical/OG URLs
  - `static/robots.txt` sitemap URL
  - JSON-LD `url` fields
  - `static/llms.txt` canonical links
  - OG image absolute URLs

## Metadata + OG images

- Every public indexable route must have a unique `title` and `description` via the SEO component.
- If you add a new public page, add `<SEO title="..." description="..." />` with unique values.
- If you change the OG image path, verify the file exists in `static/` and is accessible at the expected URL.
- The SEO component (`src/components/SEO.tsx`) handles all `<head>` meta tags via `react-helmet`. Do not add duplicate meta tags outside this component.

## Indexing controls

- Private, authenticated, personalized, and post-checkout routes must stay `noindex`.
- Current non-indexable route patterns:
  - `/payment-success`
- Keep those routes out of the sitemap (`gatsby-config.ts` → `gatsby-plugin-sitemap` → `excludes`).
- Keep crawl disallows in sync in `static/robots.txt`.
- Use the `noIndex` prop on the SEO component to emit `<meta name="robots" content="noindex, nofollow">`.

## Sitemap / robots

- `static/robots.txt` must always point at the canonical sitemap on the canonical host.
- The sitemap is generated by `gatsby-plugin-sitemap` at build time.
- Keep the sitemap limited to index-worthy public routes only.
- If you add or remove public routes, verify the sitemap excludes list is still correct.

## Structured data (JSON-LD)

- If page meaning changes, update the relevant JSON-LD schema too.
- JSON-LD is added via `react-helmet` `<script type="application/ld+json">` tags (or via the `JsonLd` helper component).
- Expected schemas:
  - Home: `Organization`, `WebSite`, `SoftwareApplication`
  - Blog posts: `BlogPosting` + `BreadcrumbList`
  - Glossary index: `DefinedTermSet`
  - Glossary terms: `DefinedTerm` + `BreadcrumbList`
  - Actions: `TechArticle` + `BreadcrumbList`
  - Feature pages: `Article` + `BreadcrumbList`
  - Guide pages: `HowTo` or `Article` + `BreadcrumbList`

## Checks

- Sanity-check:
  - `GET /robots.txt` → 200, contains sitemap URL
  - `GET /sitemap-index.xml` → 200, contains only intended public routes
  - `GET /og.png` → 200 (or equivalent OG image)
  - View source of at least one page → has canonical link, OG tags, JSON-LD
  - `/payment-success` → renders `noindex` robots meta
  - Build succeeds: `npm run build`
